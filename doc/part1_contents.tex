\subsection{Philosophy}
\begin{frame}
    \frametitle{Philosophy}
    \vspace{5mm}
    \begin{itemize}
      \item Scripting language.
      \item Emphasis on code readability.
      \item Execution speed is not a priority.
      \item Language simplicity is a priority.
      \item Readability is a priority.
      \item Blocks are defined through indentation.
    \end{itemize}
    
    \pause
    \vspace{5mm}
    Zen of Python \url{https://en.wikipedia.org/wiki/Zen\_of\_Python}.
    
    \pause
    \vspace{5mm}
    Quoting Monty Python is officially considered good style.

\subsection{Characteristics}
\begin{frame}
    \frametitle{Characteristics}
    \vspace{5mm}
    
    \begin{itemize}
      \descitem{Interpreted:}
        \begin{itemize}
          \item C: compiler + code $\rightarrow$ code $\rightarrow$ execution
          \item Java: compiler + code $\rightarrow$ bytecode; bytecode + interpreter $\rightarrow$ execution
          \item Python: interpreter + code $\rightarrow$ execution
        \end{itemize}
      \descitem{Garbage Collected:} Makes use of automatic memory management (no need to keep tract of when to \textsl{free} individual allocations).
      \descitem{Dynamically Typed:} Type safety is verified at runtime.
      \descitem{Late Binding:} Names are looked up at runtime.
      \descitem{High-Level:} Focus is on serving as a good language for abstract (as in: machine distant) operations.
    \end{itemize}
\end{frame}

\subsection{Bad Matches}
\begin{frame}
    \frametitle{Bad Matches}
    \vspace{5mm}
    
    \pause
    \begin{itemize}
      \item The use of a global interpreter lock (GIL) makes the language a bad fit for most problems that would benefit from concurrency.
      \pause
      \item The lack of \say{compile-time} type checking/safely makes the language a bad fit for any problem that requires robustness or is long-running.
      \pause
      \item The level of abstraction makes python a bad fit for any computationally heavy operations.
      \pause
          \begin{itemize}
              \item \textbf{Note:} Python code can call C, Fortran or CUDA code which can do the heavy lifting. This is what \texttt{numpy} does.
          \end{itemize}
    \end{itemize}
\end{frame}
